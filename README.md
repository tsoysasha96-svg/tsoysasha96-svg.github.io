# КодБлог

Статический блог о программировании (RU), без сборки — чистый HTML/CSS/JS. Хостится бесплатно на GitHub Pages.

## Структура

- `index.html` — главная, список статей (подтягивает `posts.json`)
- `posts/*.html` — отдельные статьи
- `posts.json` — метаданные статей для главной страницы
- `about.html`, `privacy.html` — обязательные страницы для AdSense
- `_post-template.html` — шаблон для новой статьи (сам не публикуется)
- `.nojekyll` — отключает обработку Jekyll на GitHub Pages (нужен, т.к. есть файлы с `_`)

## Как опубликовать сайт (сделать один раз)

1. Зайдите на github.com под своим аккаунтом (`tsoysasha96-svg`) и создайте **новый пустой репозиторий** с названием ровно `tsoysasha96-svg.github.io` (без README, без .gitignore — просто пустой репозиторий). Это специальное имя для "user site" — GitHub Pages подхватит его автоматически.
2. Вернитесь в терминал в этой папке и попросите меня запушить код — я подключу `origin` и сделаю `git push`. При первом push Windows откроет окно авторизации GitHub — войдите под своим аккаунтом.
3. В репозитории на GitHub: **Settings → Pages → Source: Deploy from a branch → Branch: main / (root)**. Сохраните.
4. Через 1-2 минуты сайт будет доступен по адресу `https://tsoysasha96-svg.github.io`.

## Как подключить рекламу (Google AdSense)

Это должны сделать вы сами — регистрация требует ваших личных/налоговых данных, которые я не ввожу за вас.

1. Зарегистрируйтесь на [google.com/adsense](https://www.google.com/adsense) с сайтом `https://tsoysasha96-svg.github.io`.
2. Google выдаст тег вида `<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXXXXXXXX">` — пришлите его мне (или вставьте сами) в `<head>` файла `index.html` и во все файлы в `posts/`.
3. После одобрения (обычно от нескольких дней до пары недель) замените плейсхолдеры `<div class="ad-slot">Рекламный блок</div>` на реальный код рекламного блока из кабинета AdSense.
4. Создайте в корне файл `ads.txt` с содержимым, которое вам даст AdSense (вида `google.com, pub-XXXXXXXXXX, DIRECT, f08c47fec0942fa0`) — без этого файла реклама может не показываться.
5. **Важно:** первое время у сайта не будет трафика — это нормально. AdSense физически не одобряет сайты без реального содержимого, поэтому 3 стартовые статьи и дальнейшее регулярное пополнение важны не только для читателей, но и для одобрения.

## Автоматическое пополнение сайта

Настроена периодическая задача (см. `SCHEDULED_TASK.md`), которая добавляет новую статью на выбранную тему, обновляет `posts.json` и `sitemap.xml`, коммитит и пушит изменения — без вашего участия. Раз в некоторое время стоит всё же просматривать новые статьи глазами.

## Локальный просмотр

Файлы статические, можно просто открыть `index.html` в браузере, либо поднять любой локальный сервер, например:

```
python -m http.server 8080
```

и открыть `http://localhost:8080`.
